## Step 4 - Gate المراجعة النهائية الإلزامية (قبل إغلاق المرحلة)
- قائمة تحقق إلزامية:
  - Functional: كل السيناريوهات الحرجة تعمل من الواجهة حتى قاعدة البيانات.
  - Security: تحقق صلاحيات backend وعدم الاعتماد على الواجهة للحماية.
  - Data integrity: لا كسر لعلاقات البيانات أو تاريخ العمليات.
  - Admin workflow: التدفقات الإدارية مفهومة وقابلة للمراجعة والتتبع.
  - Integration: ترابط صحيح بين modules (custody/payments/accounting/audit/settings).
- تنفيذ التحقق:
  - Backend: اختبارات مركزة للملفات المتأثرة + `pytest -m guardrail`.
  - Frontend: `npm run build`.
  - E2E/smoke انتقائي للسيناريو المتأثر عند الحاجة.
- شرط الإغلاق:
  - لا تُغلق المرحلة حتى نجاح التحقق بالكامل أو توثيق المخاطر وخطة معالجتها واعتمادها.
